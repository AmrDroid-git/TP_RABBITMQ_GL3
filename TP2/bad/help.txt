Since we have two branches sending the same type of data (Sales) to one destination (Head Office), the simplest and most effective structure is:
--> A Single "Direct" Exchange with One Shared Queue





To start all three databases, simply open your terminal in that folder and run:
docker-compose up -d


# Run BO1 SQL file
docker exec -i branch_office_1_db mysql -uroot -proot bo1_sales_db < bio1.sql

# Run BO2 SQL file
docker exec -i branch_office_2_db mysql -uroot -proot bo2_sales_db < bio2.sql

# Run HO SQL file
docker exec -i head_office_db mysql -uroot -proot ho_sales_db < ho.sql




then verify:
# Check BO1
docker exec branch_office_1_db mysql -uroot -proot bo1_sales_db -e "SELECT * FROM product_sales;"

# Check BO2
docker exec branch_office_2_db mysql -uroot -proot bo2_sales_db -e "SELECT * FROM product_sales;"

# Check HO
docker exec head_office_db mysql -uroot -proot ho_sales_db -e "SELECT * FROM product_sales;"



HOW TO OPEN THE DATABASE SHELL FROM CMD
docker exec -it branch_office_1_db mysql -uroot -proot bo1_sales_db
docker exec -it branch_office_2_db mysql -uroot -proot bo2_sales_db
docker exec -it head_office_db mysql -uroot -proot ho_sales_db

then you can try for example:
SELECT * FROM product_sales;
